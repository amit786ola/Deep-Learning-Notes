1.5 Interview Questions — Day 1
Q1: What is the difference between ML and DL?

ML uses manually engineered features; DL learns features automatically via multiple layers. DL excels at unstructured data (images, text, audio) and large datasets.

Q2: Why did Deep Learning explode after 2012?

Three factors: massive labeled datasets (ImageNet), GPU computing power, and algorithmic improvements (ReLU activation, Dropout regularization). AlexNet demonstrated this combination dramatically.

Q3: When would you NOT use deep learning?

For small/tabular datasets, when interpretability matters (medical diagnosis explanation), limited compute, or when classical ML (XGBoost) performs equally well with less complexity.

Q4: What is the Universal Approximation Theorem?

A neural network with a single hidden layer containing a sufficient number of neurons can approximate any continuous function to any desired precision. This is the theoretical justification for why neural networks work.



📅 DAY 2 — McCulloch–Pitts (MP) Neuron
2.1 Biological Inspiration
Biological Neuron          →   Artificial Neuron
--------------------------------------------------
Dendrites (receive signals) →  Inputs (x1, x2, ..., xn)
Synapse (connection weight) →  Weights (w1, w2, ..., wn)
Cell body (process)         →  Summation + Activation
Axon (output signal)        →  Output (y)

2.2 MP Neuron Model (1943)
Proposed by Warren McCulloch (neurophysiologist) and Walter Pitts (mathematician).
Architecture
Inputs: x1, x2, ..., xn  (binary: 0 or 1)
        ↓
    Aggregation: g(x) = x1 + x2 + ... + xn
        ↓
    Thresholding: y = 1 if g(x) >= θ, else 0
Mathematical Formula
g(x) = Σ xi  (sum of binary inputs)

y = f(g(x)) = { 1  if g(x) >= θ (threshold)
              { 0  otherwise
Key Properties

Inputs: Binary only (0 or 1)
Weights: All equal (no differentiation between inputs)
Threshold (θ): Hardcoded, NOT learned
Output: Binary (0 or 1)
Excitatory inputs: Normal inputs (promote firing)
Inhibitory inputs: If any inhibitory input = 1, neuron does NOT fire regardless of others


2.3 Implementing Logic Gates with MP Neuron
AND Gate (θ = 2)
x1x2g(x)y (θ=2)0000011010101121
OR Gate (θ = 1)
x1x2g(x)y (θ=1)0000011110111121
NOT Gate (use inhibitory input, θ = 0)

If x=0 → neuron fires → y=1
If x=1 → inhibitory → neuron doesn't fire → y=0

XOR Gate — CANNOT be implemented!

XOR is NOT linearly separable → MP neuron FAILS → this was the key limitation.


2.4 Limitations of MP Neuron

Binary inputs only — cannot handle real-valued inputs
No learning — threshold θ is hardcoded (not learned from data)
Equal weights — cannot differentiate importance of inputs
Not trainable — no algorithm to update parameters
Only linearly separable functions can be modeled


2.5 Code — MP Neuron
pythonimport numpy as np

class MPNeuron:
    def __init__(self, threshold):
        self.threshold = threshold  # θ — manually set, NOT learned
    
    def predict(self, X):
        """
        X: 2D array of binary inputs (n_samples, n_features)
        Returns: binary predictions (0 or 1)
        """
        # Sum all inputs for each sample
        aggregate = np.sum(X, axis=1)
        # Apply threshold function
        return (aggregate >= self.threshold).astype(int)

# Example: AND gate
X = np.array([[0,0], [0,1], [1,0], [1,1]])
y = np.array([0, 0, 0, 1])  # AND truth table

model = MPNeuron(threshold=2)
predictions = model.predict(X)
print(predictions)  # [0 0 0 1] — correct!

# Find best threshold (brute force — since no learning)
best_threshold = None
best_accuracy = 0

for theta in range(0, X.shape[1] + 1):
    model = MPNeuron(threshold=theta)
    preds = model.predict(X)
    accuracy = np.mean(preds == y)
    if accuracy > best_accuracy:
        best_accuracy = accuracy
        best_threshold = theta

print(f"Best threshold: {best_threshold}, Accuracy: {best_accuracy}")

2.6 Interview Questions — Day 2
Q1: What are the main limitations of the MP Neuron?

It accepts only binary inputs, uses equal weights, has a hardcoded (non-learnable) threshold, and can only model linearly separable functions. It cannot solve XOR.

Q2: Why can't MP Neuron solve XOR?

XOR is not linearly separable — no single straight line can separate class 0 and class 1 points. MP Neuron (like Perceptron) can only create linear decision boundaries.

Q3: What is the role of the threshold in MP Neuron?

The threshold θ determines when the neuron fires. If the sum of inputs ≥ θ, output is 1; otherwise 0. It's analogous to the biological "action potential" threshold.



📅 DAY 3 — Perceptron — Architecture & Demo
3.1 What is a Perceptron?
Proposed by Frank Rosenblatt in 1958 at Cornell. First trainable single-layer neural network.
Improvements over MP Neuron:

✅ Accepts real-valued inputs (not just binary)
✅ Has learnable weights w1, w2, ..., wn
✅ Has a learnable bias term b
✅ Uses a learning algorithm to update weights
❌ Still produces only binary output (0 or 1)
❌ Still can only solve linearly separable problems


3.2 Perceptron Architecture
       x1 ──(w1)──┐
       x2 ──(w2)──┤
       x3 ──(w3)──┼──→ Σ(wixi + b) ──→ Step Function ──→ y (0 or 1)
       ...         │
       xn ──(wn)──┘
Mathematical Formula
z = w1*x1 + w2*x2 + ... + wn*xn + b
  = W·X + b   (dot product form)

y = f(z) = { 1  if z >= 0
           { 0  if z < 0
Where:

W = weight vector [w1, w2, ..., wn]
X = input vector [x1, x2, ..., xn]
b = bias (shifts decision boundary)
f = step function (Heaviside function)

Role of Weights

Large positive weight → input strongly promotes firing
Large negative weight → input strongly inhibits firing
Weight near 0 → input has little influence

Role of Bias

Allows the decision boundary to be shifted
Without bias: decision boundary always passes through origin
With bias: more flexible positioning of boundary


3.3 Geometric Interpretation
The equation W·X + b = 0 defines a hyperplane (line in 2D, plane in 3D):

Points on one side → output = 1
Points on the other side → output = 0

In 2D:
     w1*x1 + w2*x2 + b = 0
     → Decision boundary is a LINE

     Class 1: above the line (z ≥ 0)
     Class 0: below the line (z < 0)

3.4 Perceptron Learning Algorithm
Goal: Find weights W and bias b such that predictions match labels.
Initialization:
  W = [0, 0, ..., 0]  (or small random values)
  b = 0

For each epoch:
  For each training sample (xi, yi):
    ŷ = predict(xi)     # forward pass
    if ŷ != yi:          # if wrong prediction
      Update weights:
        W = W + η * yi * xi    (if yi=1: add xi; if yi=0: subtract)
        b = b + η * yi         (bias update)
Where η (eta) = learning rate (typically 0.01–0.1).
Perceptron Convergence Theorem

If the data is linearly separable, the Perceptron algorithm is guaranteed to converge (find correct weights) in a finite number of steps.


3.5 Code — Perceptron from Scratch
pythonimport numpy as np
from sklearn.datasets import make_classification
from sklearn.model_selection import train_test_split

class Perceptron:
    def __init__(self, learning_rate=0.01, n_epochs=1000):
        self.lr = learning_rate
        self.n_epochs = n_epochs
        self.weights = None
        self.bias = None
    
    def fit(self, X, y):
        n_samples, n_features = X.shape
        # Initialize weights and bias to zero
        self.weights = np.zeros(n_features)
        self.bias = 0
        
        for epoch in range(self.n_epochs):
            for i in range(n_samples):
                # Forward pass
                z = np.dot(X[i], self.weights) + self.bias
                y_pred = 1 if z >= 0 else 0
                
                # Update rule (only if wrong)
                if y_pred != y[i]:
                    # y[i]=1 and pred=0: add xi to weights
                    # y[i]=0 and pred=1: subtract xi from weights
                    update = self.lr * (y[i] - y_pred)
                    self.weights += update * X[i]
                    self.bias += update
    
    def predict(self, X):
        z = np.dot(X, self.weights) + self.bias
        return (z >= 0).astype(int)
    
    def accuracy(self, X, y):
        return np.mean(self.predict(X) == y)

# Test on synthetic data
X, y = make_classification(n_samples=200, n_features=2,
                            n_redundant=0, n_informative=2,
                            random_state=42)
X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.2)

model = Perceptron(learning_rate=0.1, n_epochs=100)
model.fit(X_train, y_train)
print(f"Test Accuracy: {model.accuracy(X_test, y_test):.4f}")
print(f"Weights: {model.weights}")
print(f"Bias: {model.bias}")

3.6 Interview Questions — Day 3
Q1: How does Perceptron improve upon MP Neuron?

Perceptron accepts real-valued inputs, has learnable weights and bias, and uses a training algorithm (Perceptron Learning Rule) to adjust parameters from data.

Q2: What is the role of the bias term?

Bias shifts the decision boundary so it doesn't have to pass through the origin. It gives the model more flexibility. Mathematically, bias is equivalent to adding a constant input x0=1 with weight b.

Q3: What happens if data is not linearly separable?

The Perceptron will never converge. It will keep updating weights forever without finding a solution. This is why it fails on XOR. Solution: use multi-layer networks.

Q4: What is the Perceptron Convergence Theorem?

If training data is linearly separable, the Perceptron algorithm will find a separating hyperplane in a finite number of iterations. The bound depends on the margin between classes.



📅 DAY 4 — Perceptron Trick (Geometric Intuition)
4.1 The Perceptron Trick — Geometric View
The Perceptron Trick explains geometrically WHY the weight update rule works.
Setup

Decision boundary: w1*x1 + w2*x2 + b = 0
A misclassified point: p = (x1, x2) with label y

Case 1: Point should be POSITIVE (y=1) but classified NEGATIVE
The point p is on the wrong side. We want to move the line towards p.
W_new = W_old + η * x   (add the point's coordinates)
b_new = b_old + η * 1

Effect: The decision boundary "rotates/shifts" towards the misclassified point.
Case 2: Point should be NEGATIVE (y=0) but classified POSITIVE
The point p is on the wrong side. We want to move the line away from p.
W_new = W_old - η * x   (subtract the point's coordinates)
b_new = b_old - η * 1

Effect: The decision boundary "rotates/shifts" away from the misclassified point.

4.2 Why Does This Work?
After adding the point coordinates to weights:
W_new · p = (W_old + η*p) · p
           = W_old·p + η*|p|²

Since |p|² > 0, the dot product increases,
meaning the point is closer to the positive side.
This mathematically proves the update rule moves the boundary in the right direction.

4.3 Visualizing the Perceptron Trick
Before update:
      +  |          ← decision boundary (wrong)
    + +  | *         * = misclassified point (should be +)
         |

After update (line moved towards *):
      +  *  |       ← new decision boundary
    + +     |

4.4 Code — Visualizing Perceptron Learning
pythonimport numpy as np
import matplotlib.pyplot as plt

def plot_decision_boundary(X, y, weights, bias, title=""):
    """Plot data points and perceptron decision boundary"""
    plt.figure(figsize=(8, 6))
    
    # Plot data points
    plt.scatter(X[y==1, 0], X[y==1, 1], c='blue', label='Class 1', s=50)
    plt.scatter(X[y==0, 0], X[y==0, 1], c='red', label='Class 0', s=50)
    
    # Plot decision boundary: w1*x1 + w2*x2 + b = 0
    # → x2 = -(w1*x1 + b) / w2
    x1_range = np.linspace(X[:, 0].min()-1, X[:, 0].max()+1, 100)
    if weights[1] != 0:
        x2_boundary = -(weights[0] * x1_range + bias) / weights[1]
        plt.plot(x1_range, x2_boundary, 'k-', label='Decision Boundary')
    
    plt.legend()
    plt.title(title)
    plt.xlabel('x1')
    plt.ylabel('x2')
    plt.show()

# Perceptron with visualization
class PerceptronViz:
    def __init__(self, lr=0.1):
        self.lr = lr
        self.weights = None
        self.bias = None
        self.history = []  # store weight history
    
    def fit(self, X, y, n_epochs=10):
        self.weights = np.zeros(X.shape[1])
        self.bias = 0
        
        for epoch in range(n_epochs):
            errors = 0
            for i in range(len(X)):
                z = np.dot(X[i], self.weights) + self.bias
                y_pred = 1 if z >= 0 else 0
                
                if y_pred != y[i]:
                    errors += 1
                    # The Perceptron Trick
                    update = self.lr * (y[i] - y_pred)
                    self.weights += update * X[i]
                    self.bias += update
                    # Save state after each update
                    self.history.append((self.weights.copy(), self.bias))
            
            print(f"Epoch {epoch+1}: Errors = {errors}")
            if errors == 0:
                print("Converged!")
                break

4.5 Interview Questions — Day 4
Q1: Explain the Perceptron Trick geometrically.

If a point is misclassified as negative (but should be positive), we add that point's coordinates to the weight vector. This rotates the decision boundary toward the misclassified point. If misclassified as positive, we subtract. This ensures the boundary progressively corrects itself.

Q2: What is the learning rate and why does it matter?

Learning rate η controls the step size of weight updates. Too small → slow convergence. Too large → oscillation, may never converge. Typical range: 0.001 to 0.1.

Q3: Is the Perceptron deterministic?

The convergence is guaranteed for linearly separable data, but the final weights depend on the order of training examples and the learning rate. Different orderings may yield different but equally valid hyperplanes.



📅 DAY 5 — Perceptron Loss Function
5.1 Why Do We Need a Loss Function?
The Perceptron Learning Rule updates on misclassified points, but it doesn't optimize a clearly defined objective. Reformulating as loss minimization makes it:

Mathematically principled
Extendable to more complex models
Connectable to gradient-based optimization


5.2 Perceptron Loss (Hinge-like)
For a misclassified sample:
Loss = -y * (W·x + b)

Where:
  y ∈ {-1, +1}  (labels in ±1 convention)
  W·x + b = signed distance from boundary
If correctly classified: y * (W·x + b) > 0 → contribution to loss = 0
If misclassified: y * (W·x + b) < 0 → positive loss
Total Perceptron Loss
L(W, b) = - Σ  y_i * (W·x_i + b)
         (over misclassified points only)

5.3 Gradient of Perceptron Loss
∂L/∂W = - Σ y_i * x_i    (over misclassified points)
∂L/∂b = - Σ y_i

Gradient Descent update:
  W ← W - η * ∂L/∂W = W + η * Σ y_i * x_i
  b ← b - η * ∂L/∂b = b + η * Σ y_i
This is exactly the Perceptron update rule! The trick simply IS gradient descent on this loss.

5.5 Interview Questions — Day 5
Q1: Why is the Perceptron loss not differentiable everywhere?

The step function (Heaviside) is not differentiable at z=0, and has zero gradient everywhere else. This makes gradient descent ineffective with the step function — which is why we need the Sigmoid neuron.

Q2: What is the connection between the Perceptron update rule and gradient descent?

The Perceptron update rule is exactly gradient descent applied to the Perceptron loss function L = -Σ y_i*(W·x_i + b) over misclassified points. This unifies the geometric trick with optimization theory.

Q3: What is a surrogate loss?

When the true loss (e.g., 0-1 loss = misclassification rate) is non-differentiable, we use a smooth approximation (surrogate) like cross-entropy or hinge loss that we can optimize with gradient descent.



📅 DAY 6 — Sigmoid Neuron
6.1 Motivation: Why Sigmoid?
Problem with Perceptron:

Step function has zero gradient almost everywhere
Cannot use gradient descent effectively
A tiny change in weights → output flips from 0 to 1 abruptly
No smooth learning signal

Sigmoid Neuron solves this by:

Using a smooth, differentiable activation function
Outputting a probability between 0 and 1
Enabling gradient-based learning


6.2 Sigmoid Neuron Architecture
       x1 ──(w1)──┐
       x2 ──(w2)──┤
       x3 ──(w3)──┼──→ z = Σ(wixi + b) ──→ σ(z) ──→ ŷ ∈ (0, 1)
       ...         │
       xn ──(wn)──┘
Sigmoid (Logistic) Function
σ(z) = 1 / (1 + e^(-z))
Properties:

Output always in (0, 1) — interpreted as probability
σ(0) = 0.5
σ(+∞) → 1, σ(-∞) → 0
Smooth, differentiable everywhere
Derivative: σ'(z) = σ(z) * (1 - σ(z))  ← very important for backprop!
